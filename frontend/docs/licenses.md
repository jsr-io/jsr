title: Licenses
description: Information about licenses supported on JSR.
---

JSR requires all packages to have a valid license. This ensures legal clarity
for package users and maintainers.

## License Requirements

All packages published to JSR must specify a license in one of two ways:

1. **Configuration file**: Add a `license` field to your `jsr.json` or
   `deno.json` with an [SPDX license identifier](https://spdx.org/licenses/)
2. **License file**: Include a license file in your package root (see supported
   filenames below)

### Supported License Files

JSR recognizes these license file names in your package root:

- `LICENSE`
- `LICENSE.md`
- `LICENSE.txt`
- `LICENCE`
- `LICENCE.md`
- `LICENCE.txt`

### Configuration Example

```json
{
  "name": "@scope/package",
  "version": "1.0.0",
  "license": "MIT"
}
```
